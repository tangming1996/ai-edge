== Next Steps ==

1. Verify the gateway row was created automatically (the chart sets
   GATEWAY_AUTO_REGISTER=true on the DaemonSet, so each gateway node
   calls GatewayService.CreateGateway on first boot, using the K8s
   node name as the gateway NAME). You can list gateways without
   leaving the cluster:

   $ kubectl exec -n {{ .Release.Namespace }} deploy/{{ include "ai-edge.fullname" . }}-apiserver -- \
     edgectl gateway list

   If GATEWAY_AUTO_REGISTER is disabled, register the gateway explicitly:

   $ kubectl exec -n {{ .Release.Namespace }} deploy/{{ include "ai-edge.fullname" . }}-apiserver -- \
     edgectl gateway register --name <gateway-name> --region <region>
   # Last line is always: gateway_id: <uuid>

2. (Optional, post-register) Backfill the per-gateway business
   attributes (region / endpoint / labels). Self-registration only
   writes the NAME (= node name); use `edgectl gateway update` to
   fill in the rest:

   $ kubectl exec -n {{ .Release.Namespace }} deploy/{{ include "ai-edge.fullname" . }}-apiserver -- \
     edgectl gateway update <gateway-name> \
       --region cn-east-1 \
       --endpoint gateway-shanghai-01.example.com:9443

3. Create bootstrap tokens for node onboarding (use the gateway_id
   from step 1):

   $ kubectl exec -n {{ .Release.Namespace }} deploy/{{ include "ai-edge.fullname" . }}-apiserver -- \
     edgectl token create --gateway <gateway_id> --expires-in 24h

4. Label the nodes that should run gateway-runtime (the node label
   is the only node-side configuration that matters now; the old
   `edgeai.io/gateway-*` annotations are no longer read by the
   DaemonSet):

   $ kubectl label node <node-name> node.edgeai.io/role=gateway

5. Install edge-agent on edge nodes (run on each node):

   $ curl -sL https://raw.githubusercontent.com/tangming1996/ai-edge/main/manifests/scripts/install-edge-agent.sh | \
       GATEWAY_ID=<gateway_id> \
       GATEWAY_ADDR={{ include "ai-edge.fullname" . }}-gateway-runtime.{{ .Release.Namespace }}.svc.cluster.local:{{ .Values.gatewayRuntime.service.grpcPort }} \
       TOKEN=<bootstrap-token> \
       bash

6. Install edgectl on your management machine:

   $ curl -sL https://raw.githubusercontent.com/tangming1996/ai-edge/main/manifests/scripts/install-edgectl.sh | bash

== Database migrations ==

Database schema migrations run automatically as a pre-install,pre-upgrade
Helm hook Job ({{ include "ai-edge.fullname" . }}-migrate). The chart ships
every migrations/*.sql file as a ConfigMap; the Job runs `migrate -path
/migrations -database $DATABASE_URL up` against {{ if .Values.postgresql.enabled }}the bundled PostgreSQL{{ else }}{{ .Values.db.host }}{{ end }} and Helm
blocks the rest of the release until it reports Complete. Inspect the log with:

  $ kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/component=migration --tail=200

To opt out, set `--set migration.enabled=false` and run migrations out of band.

== Required Pre-requisites ==

- A database reachable from the cluster. Either:
    * enable `postgresql.enabled=true` (the chart ships Postgres and auto-generates a password), or
    * set `db.host` + `db.existingSecret` for an external DB.
- A CA the apiserver uses to sign bootstrap / leaf certs. Either:
    * set `apiserver.ca.generate=true` (self-signed, 10-year validity), or
    * set `apiserver.ca.existingSecret=...` for your own CA bundle.
- An mTLS server cert the gateway presents to edge-agents. Either:
    * set `gatewayRuntime.tls.generate=true` (signed by the apiserver CA when
      both are auto-generated, otherwise self-signed), or
    * set `gatewayRuntime.tls.existingSecret=...` to bring your own.

For full guidance see docs/helm-secrets.md.
