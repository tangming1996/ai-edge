== Next Steps ==

1. Create bootstrap tokens for node onboarding:
   $ kubectl exec -n {{ .Release.Namespace }} deploy/{{ include "ai-edge.fullname" . }}-apiserver -- \
     edgectl token create --gateway <gateway-id> --expires-in 24h

2. Label gateway nodes:
   $ kubectl label node <node-name> node.edgeai.io/role=gateway

3. Install edge-agent on edge nodes (run on each node):
   $ curl -sL https://raw.githubusercontent.com/edgeai-platform/ai-edge/main/manifests/scripts/install-edge-agent.sh | \
       GATEWAY_ID=<gateway-id> \
       CONTROL_PLANE_ADDR={{ include "ai-edge.fullname" . }}-apiserver.{{ .Release.Namespace }}.svc.cluster.local:443 \
       TOKEN=<bootstrap-token> \
       bash

4. Install edgectl on your management machine:
   $ curl -sL https://raw.githubusercontent.com/edgeai-platform/ai-edge/main/manifests/scripts/install-edgectl.sh | bash

== Required Pre-requisites ==

- PostgreSQL database (set `db.host` if external, or enable `postgresql.enabled=true`)
- TLS certificates for apiserver and gateway-runtime (mounted via secrets referenced in `apiserver.caSecretName` and `gatewayRuntime.gatewayTlsSecretName`)
